## 1. Tool-Uebersicht

### Was ist n8n?

n8n (ausgesprochen "n-eight-n", abgeleitet von "nodemation" = Node + Automation) ist eine **Fair-Code-lizenzierte Workflow-Automatisierungsplattform**, die die Geschwindigkeit von No-Code mit der Flexibilitaet von Full-Code-Entwicklung vereint. Die Plattform ermoeglicht es, Geschaeftsprozesse visuell zu gestalten, KI-Agenten zu bauen und komplexe Automatisierungen ueber hunderte von Applikationen hinweg zu orchestrieren.

Im Kern ist n8n ein **node-basierter visueller Editor**, in dem jeder Schritt eines Workflows als "Node" dargestellt wird. Diese Nodes koennen Trigger, Datenverarbeitungen, API-Aufrufe, KI-Modelle oder benutzerdefinierte Code-Bloecke sein.

**Offizielle Beschreibung (2026):** "Fair-code workflow automation platform with native AI capabilities. Combine visual building with custom code, self-host or cloud, 400+ integrations."

### Open-Source vs. Cloud

n8n bietet zwei Bereitstellungsmodelle:

**1. Self-Hosted (Community Edition) -- KOSTENLOS**
- Vollstaendig quelloffener Code (Fair-Code-Lizenz, aehnlich Open Source)
- Unbegrenzte Workflow-Ausfuehrungen
- Alle 400+ Integrationen enthalten
- Volle Kontrolle ueber Daten und Infrastruktur
- Einzige Kosten: eigene Infrastruktur (typischerweise 5-20 EUR/Monat)
- Ideal fuer DSGVO-Compliance und Datensouveraenitaet

**2. n8n Cloud (Managed Service)**
- Von n8n gehosteter Service
- EU-Rechenzentrum in Frankfurt
- Automatische Updates und Wartung
- Verschiedene Preisstufen basierend auf Ausfuehrungen

## 2. Key Features & Capabilities

### 2.1 Visueller Workflow Builder

Der Kern von n8n ist ein **drag-and-drop Workflow-Editor** im Browser:

- **Node-basierte Architektur:** Jeder Workflow besteht aus miteinander verbundenen Nodes
- **Visuelles Debugging:** Echtzeit-Ausfuehrungsdaten an jedem Node sichtbar
- **Branching & Conditional Logic:** If/Else, Switch, Merge, Loops
- **Sub-Workflows:** Workflows koennen andere Workflows aufrufen
- **Draft/Published-System (neu in 2.0):** Aenderungen koennen vorbereitet und separat veroeffentlicht werden
- **Version History:** Aenderungsverlauf fuer jeden Workflow
- **Collaboration:** Mehrere Nutzer koennen an Workflows arbeiten (Role-Based Access)

### 2.2 400+ Integrationen (Nodes)

n8n bietet **ueber 400 native Integrationen** (teilweise werden 500+ genannt) und kann ueber generische Nodes praktisch jede API ansprechen:

**Wichtige Kategorien:**
- **CRM:** Salesforce, HubSpot, Pipedrive, Zoho CRM
- **Communication:** Slack, Microsoft Teams, Discord, Telegram, WhatsApp
- **E-Mail:** Gmail, Outlook, SMTP/IMAP, Mailchimp, SendGrid
- **Datenbanken:** PostgreSQL, MySQL, MongoDB, Redis, Supabase, Airtable
- **Cloud Storage:** Google Drive, Dropbox, OneDrive, AWS S3
- **Projektmanagement:** Jira, Asana, Trello, Notion, ClickUp, Monday.com
- **E-Commerce:** Shopify, WooCommerce, Stripe, PayPal
- **Social Media:** LinkedIn, Twitter/X, Facebook, Instagram
- **ERP/Buchhaltung:** SAP, DATEV (ueber API), Lexoffice, SevDesk
- **Generische Nodes:** HTTP Request, GraphQL, FTP, SSH, MQTT

**Entscheidend:** Fuer jede Applikation ohne nativen Node gibt es den **HTTP Request Node**, der beliebige REST-APIs ansprechen kann. Damit ist n8n de facto unbegrenzt erweiterbar.

### 2.3 Custom Node Development

- **Eigene Nodes** koennen in TypeScript/JavaScript entwickelt werden
- **Community Nodes:** Marktplatz fuer von der Community entwickelte Integrationen
- n8n stellt ein **CLI-Tool und Boilerplate** fuer die Node-Entwicklung bereit
- Best Practice: try/catch-Bloecke, `NodeOperationError`/`NodeApiError`-Klassen
- Bluebatch kann **massgeschneiderte Nodes** fuer Kunden entwickeln -- ein starkes Differenzierungsmerkmal

### 2.4 KI/LLM-Integration (Grosse Staerke in 2026)

n8n hat sich als **fuehrende Plattform fuer KI-Workflow-Automatisierung** positioniert:

- **AI Agent Node:** Autonome KI-Agenten, die Entscheidungen treffen, mit Apps interagieren und Aufgaben ausfuehren
- **LLM-agnostisch:** Unterstuetzt OpenAI (GPT-4o), Anthropic (Claude), Google Gemini, Mistral, lokale Modelle (Ollama, LM Studio)
- **LangChain-Integration:** Native Nodes fuer die gesamte LangChain-Toolchain
- **Memory Nodes:** Konversationsgedaechtnis fuer KI-Agenten
- **Vector Store Nodes:** Anbindung an Pinecone, Qdrant, Weaviate, Supabase Vector, ChromaDB
- **Tools Nodes:** KI-Agenten koennen andere Workflow-Nodes als "Tools" nutzen
- **RAG-Workflows (Retrieval Augmented Generation):** Einfach als Workflow aufbaubar
- **Self-hosted LLMs:** Moeglichkeit, lokale LLMs einzubinden -- wichtig fuer Datenschutz

> **Bluebatch Sales-Angle:** "Wir bauen KI-Agenten, die nicht nur chatten, sondern handeln -- direkt integriert in Ihre bestehenden Systeme, mit voller Kontrolle ueber Ihre Daten."

### 2.5 Code Nodes (JavaScript & Python)

- **Code Node (JavaScript):** Vollstaendiger Node.js-Zugriff fuer beliebige Logik
- **Code Node (Python):** Python-Support fuer Data-Science- und ML-Workflows
- **Expression Editor:** Inline-Ausdruecke fuer schnelle Datentransformationen
- **npm-Pakete:** Koennen in Self-Hosted-Instanzen eingebunden werden
- Ideal fuer Teams, die sowohl visuell als auch per Code arbeiten wollen

### 2.6 Error Handling, Retries & Webhooks

**Error Handling:**
- Fehlerworkflows (Error Trigger Node) fangen Fehler auf und benachrichtigen oder beheben sie
- Try/Catch-Muster auf Node-Ebene
- Continue-on-Error-Option pro Node
- Detaillierte Fehlerberichte mit Kontext (Item-Index, Response-Codes)

**Retries:**
- Automatische Wiederholungsversuche konfigurierbar pro Node
- Exponentielles Backoff unterstuetzt
- Manuelle Wiederholung fehlgeschlagener Ausfuehrungen

**Webhooks:**
- Webhook Node fuer eingehende HTTP-Anfragen
- Respond to Webhook Node fuer benutzerdefinierte HTTP-Antworten
- Unterstuetzung fuer GET, POST, PUT, DELETE, PATCH
- Header-/Body-Parsing, Authentifizierung
- Produktions- und Test-URLs getrennt

| Vorteil | Details |

## 4. Staerken vs. Schwaechen

### 4.1 Staerken (Pros)

| Staerke | Details |
|---|---|
| **Self-Hosting moeglich** | Volle Datenkontrolle, DSGVO-konform, keine Vendor-Lock-In |
| **Fair-Code / Open Source** | Quellcode einsehbar, anpassbar, keine Black Box |
| **Kosteneffizienz** | Community Edition kostenlos; Cloud 10-20x guenstiger als Zapier bei komplexen Workflows |
| **KI-First-Plattform** | Fuehrend bei KI/LLM-Integration, native AI Agent Nodes |
| **Code + No-Code** | JavaScript & Python Code Nodes fuer maximale Flexibilitaet |
| **Starke Community** | 177k+ GitHub Stars, 200k+ Community-Mitglieder, 8500+ Workflow-Templates |
| **Deutsches Unternehmen** | GmbH mit Sitz in Berlin, EU-Datenhaltung, DSGVO-Naehe |
| **Execution-basierte Abrechnung** | Pro Workflow-Run, nicht pro Schritt -- massiver Kostenvorteil bei mehrstufigen Workflows |
| **Unbegrenzte Nutzer** | Alle Plaene beinhalten unbegrenzte Nutzer -- kein Per-Seat-Pricing |
| **Enterprise-Ready (v2.0)** | SSO, RBAC, Audit Logs, Draft/Publish-System |

### 4.2 Schwaechen (Cons)

| Schwaeche | Details |
|---|---|
| **Steilere Lernkurve** | Nicht so einfach wie Zapier fuer absolute Anfaenger |
| **Technische Kompetenz noetig** | API-Verstaendnis, JSON-Kenntnisse fuer fortgeschrittene Workflows |
| **Self-Hosting erfordert DevOps** | Server-Management, Updates, Sicherheit selbst verantworten |
| **Weniger native Integrationen** | ~400-500 vs. 8.000+ bei Zapier (aber HTTP Node kompensiert) |
| **Dokumentation ausbaufaehig** | Teilweise lueckenhaft, Community-Foren als Hauptinformationsquelle |
| **Support limitiert** | Kostenloser Plan nur Community-Support; Business/Enterprise fuer Priority-Support noetig |
| **Komplexe Workflows unuebersichtlich** | Sehr grosse Workflows koennen visuell unuebersichtlich werden |
| **Kein natuerlichsprachliches Interface** | Kein "beschreibe was du willst"-Feature fuer Non-Tech-User |

### 4.3 Wann ist n8n die BESTE Wahl?

- **Datenschutz & Compliance sind Prioritaet** (DSGVO, AI Act, NIS2)
- **Hohe Workflow-Komplexitaet** (viele Schritte, Verzweigungen, Loops)
- **Hohes Ausfuehrungsvolumen** (Self-Hosted = unbegrenzte Runs)
- **KI/LLM-Workflows** gebraucht werden
- **Budget-Sensitivitaet** bei gleichzeitigem Bedarf an Leistung
- **Technisches Team** vorhanden oder externer Partner (= Bluebatch!)
- **Deutsches Unternehmen** als Anbieter gewuenscht
- **Custom Integrations** fuer Nischen-Software noetig

### 4.4 Wann ist n8n NICHT die richtige Wahl?

- **Rein nicht-technische Teams** ohne jegliche API-Erfahrung (Zapier ist einfacher)
- **Tiefe Microsoft-365-Integration** als Primaerziel (Power Automate ist hier besser)
- **Einfachste Zwei-Schritt-Automationen** (Zapier reicht und ist schneller aufgesetzt)
- **Kein Interesse an Self-Hosting** und gleichzeitig sehr kleines Budget
- **Extrem nischenhafte SaaS-Tools** ohne API, die nur Zapier-Konnektoren haben
